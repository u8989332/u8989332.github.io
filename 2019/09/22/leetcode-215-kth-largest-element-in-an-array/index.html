LeetCode 215 Kth Largest Element in an Array

Posted on 2019-09-22 Edited on 2023-11-19 In C# , LeetCode , Level-Medium , 程式解題 , ProgramSolving

LeetCode 215 Kth Largest Element in an Array 解題報告

平台：

LeetCode

題號：

215 - Kth Largest Element in an Array

題目連結：

https://leetcode.com/problems/kth-largest-element-in-an-array/

題目說明：

輸入一個整數陣列nums，求陣列中第k大的數字。

解題方法：

  1. 使用時間複雜度O(nlogn)的排序方法。
  2. 使用Heap，維護只有k個節點的Heap即可求出第k大的解，平均時間複雜度為O(nlogk)。
  3. 使用Quick Sort的原理，找第k大的數字，本篇採用使方案。依Quick Sort原理，會將陣列做部分排序，這裡從大排到小。隨機選一個點為pivot，從左邊與從右邊找出比pivot還小 / 還大 互相交換，會分成2個區塊，再判斷k位置繼續往左邊或右邊尋找。時間複雜度為O(N)。以下用圖例解釋每個步驟。

　　假設有1個輸入陣列nums為[4, 1 , 6 , 5 , 9 , 2]，left與right為最左/最右的起始位置，分別是0與5，而設置i 與 j 也分別為left 與 right。pivot的選取採用nums[left]，也就是4，而K為第2大的數字，轉成索引後變成k = 2 - 1 = 1。如圖1.

圖1

　　從左邊開始，找nums[i] <= pivot的位置；從右邊開始，找nums[j] >= pivot的位置，i與j分別停在0 與 4 的位置，並且交換，再往下個位置，如圖2.

圖2

再次找nums[i] <= pivot的位置與nums[j] >= pivot的位置，i 與 j停在 1 與 3，並且交換，再往下個位置，如圖3.

圖3

再次找nums[i] <= pivot的位置與nums[j] >= pivot的位置，i 與 j停在 3 與 2，i與j的位置已超出邊界，而k <= j ，代表k在左邊的區塊，所以將進行第2輪的排序，如圖4.

圖4

再次找nums[i] <= pivot的位置與nums[j] >= pivot的位置，i 與 j都停在1的位置，並且交換，並進到下個位置，如圖5.

圖5

此時j已經超出left，所以k不會在左邊的區塊；而k >= i，代表k將在i 與right之間，進行第3輪的排序，如圖6.

圖6

pivot目前為nums[left] = 5，再次找nums[i] <= pivot的位置與nums[j] >= pivot的位置，i 與 j停在1與2的位置，並且交換，再往下個位置，如圖7.

圖7

由於j <= left 且 i >= right，已經沒有再排序的需要，回傳nums[k] = nums[1] = 6為解.

難度為Medium
