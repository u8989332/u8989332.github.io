UVa 1368 DNA Consensus String 解題報告

平台：

UVa Online Judge

題號：

1368 - DNA Consensus String

題目連結：

https://uva.onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=24&page=show_problem&problem=4114

題目說明：

給m個只有包含T、C、G、A字母的DNA序列，長度皆為n，找出一個DNA序列是它們的 consensus string。 consensus string的定義為Hamming distance是最小的。如果有多個consensus string，回傳字典排序最小的。
Hamming Distance的計算，比如第1筆測資，

  • 字母A跟5筆DNA序列的第1個字母的Hamming Distance之和等於 1 + 1 + 0 + 1 + 1 = 4
  • 字母C跟5筆DNA序列的第1個字母的Hamming Distance之和等於 1 + 1 + 1 + 1 + 1 = 5
  • 字母G跟5筆DNA序列的第1個字母的Hamming Distance之和等於 1 + 1 + 1 + 1 + 1 = 5
  • 字母T跟5筆DNA序列的第1個字母的Hamming Distance之和等於 0 + 0 + 1 + 0 + 0 = 1

　　所以第1筆測資的答案的第1個字母會是T，再看第2個字母

  • 字母A跟5筆DNA序列的第2個字母的Hamming Distance之和等於 0 + 0+ 0 + 1 + 0 = 1
  • 字母C跟5筆DNA序列的第2個字母的Hamming Distance之和等於 1 + 1 + 1 + 1 + 1 = 5
  • 字母G跟5筆DNA序列的第2個字母的Hamming Distance之和等於 1 + 1 + 1 + 0 + 1 = 4
  • 字母T跟5筆DNA序列的第2個字母的Hamming Distance之和等於 1 + 1 + 1 + 1 + 1 = 5

所以第1筆測資的答案的第2個字母會是A。其他以此類推。

範例輸入：

3
5 8
TATGATAC
TAAGCTAC
AAAGATCC
TGAGATAC
TAAGATGT
4 10
ACGTACGTAC
CCGTACGTAG
GCGTACGTAT
TCGTACGTAA
6 10
ATGTTACCAT
AAGTTACGAT
AACAAAGCAA
AAGTTACCTT
AAGTTACCAA
TACTTACCAA

範例輸出：

TAAGATAC
7
ACGTACGTAA
6
AAGTTACCAA
12

解題方法：

計算所有A、C、G、T與各DNA序列的每個字元最小的Hamming Distance和，可以先從T= > G => C => A判斷，最終可以求出字典最小的consensus string。時間複雜度為O(n * m)。

難度為Easy

LeetCode 23 Merge k Sorted Lists

Posted on 2019-09-01 Edited on 2023-11-19 In C# , LeetCode , 程式解題 , Level-Hard , ProgramSolving

LeetCode 23 Merge k Sorted Lists 解題報告

平台：

LeetCode

題號：

23 - Merge k Sorted Lists

題目連結：

https://leetcode.com/problems/merge-k-sorted-lists/

題目說明：

給K串Linked List，並將它們全部合併。合併的規則為每個節點的值由小排到大。

範例輸入：

[ 1->4->5, 1->3->4, 2->6 ]

範例輸出：

1->1->2->3->4->4->5->6

解題方法：

解法有暴力法與Divide&Conquer(分而治之法)，假設有K個Linked List，平均每串有N個Node。

首先分析暴力法的時間複雜度：先由第1與第2 Linked List互相比較並串新的Linked List，需要花費N + N的時間；第3個與前面新的Linked List互相比較並串新的Linked List，需要花費2N + N的時間；到最後第K個Linked List與前面新的Linked List互相比較並串新的Linked List，需要花費 (K - 1) * N + N的時間，所以全部的時間花費為  (N + N) + (2N + N) + …. + (K - 1) * N + N = (2 + K) * (K - 1) / 2 * N = O(K^2 * N)

Divide&Conquer的時間複雜度分析：將K個Linked List不斷切一半去合併，切到最小的單位（只剩2個Linked List）時，會需要K/2次的比較，每次比較會有2N個Node；合併後再進行K/4次的比較，每次比較會有4N個Node；到最後一次時；會變成K / 2^logK次的比較，每次比較會有2^logK * N個Node。所以全部的時間花費為 (K/2 * 2N) + (K/4 * 4N) + …. + (K / 2^ logK) * 2^logK * N  = KN + KN + …. + KN = O(logK * KN) 　

以時間上的比較，Divide&Conquer快了暴力法logK / K倍。採用Divide&Conquer的2串合併時，延續[解題] LeetCode - 21 Merge Two Sorted Lists的解法。實作上可參考Merge Sort或Quick Sort在切分子問題的精神。

難度為Hard

138 - Copy List with Random Pointer

題目說明：

給1串Linked List，此Linked List除了有Next指標指向下個Node之外，還有個Random指標，沒有順序的指向任何Node。要回傳一個Deep Copy的Linked List，而Deep Copy的意思為完全不同記憶體位置的複製。

範例輸入：

範例輸出：

同輸入的資料內容、Next、Random的全新Linked List

解題方法：

  1. 建立一個Map，該Map的Key為Node指標，指向舊的Linked List的Node；Value也是Node指標，指向Deep Copy的Node。
  2. 先第1次歷遍舊的Linked List，並用Map建立舊Node與新Node的Mapping。
  3. 做第2次歷遍，從Head開始，將每個舊的Node的Next與Random關係，在新的Node之間也建立關係。
  4. 最後回傳Map[Head]，代表回傳新的Linked List。

　　以下用一個Linked List說明，一開始為1->2->3的LinkedList ，但Random為1->3->2->1，如圖1. 黑色箭頭為Next、紅色箭頭為Random。
