UVa 1339 Ancient Cipher 解題報告

平台：

UVa Online Judge

題號：

1339 - Ancient Cipher

題目連結：

https://uva.onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=4085

題目說明：

給2個字串，求第1個字串是否能編碼成第2個字串。編碼規則為第1個字串的每個字元能1對1映射第2個字串的每個字元。以範例輸入第1筆測資JWPUDJSTVP，它的
J => I、W => V、P=>O….、T=>S，所以是YES。而第2筆測資MAMA只有2個不同字元，目標ROME有4個不同字元，無法完整1對1映射，所以是NO。

範例輸入：

JWPUDJSTVP
VICTORIOUS
MAMA
ROME
HAHA
HEHE
AAA
AAA
NEERCISTHEBEST
SECRETMESSAGES

範例輸出：

YES
NO
YES
YES
NO

解題方法：

不用管字元先後順序，而是將2個字串的26個英文字母字元出現數量各自加總後排序。排序後2個字元數量由小到大比對，有不同數量代表無法編碼，反之可以。
　　比如測資第3筆的HAHA和HEHE，他們字元數量排序都是{0,0,…..,0,2,2}，所以是可以編碼的。

難度為Easy

UVa 11809 Floating-Point Numbers

Posted on 2019-10-05 Edited on 2023-11-19 In C# , ACM , Level-Medium , 程式解題 , ProgramSolving , UVa

UVa 11809 Floating-Point Numbers 解題報告

平台：

UVa Online Judge

題號：

11809 - Floating-Point Numbers

題目連結：

https://uva.onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=2909

題目說明：

題目很長，經過整理的意思是

A\times10^{B} = (2^{-1}+2^{-2}+..+2^{-M-1})\times2^{2^{E}-1}

，求M與E。輸入0e0則結束程式。

範例輸入：

5.699141892149156e76
9.205357638345294e18
0e0

範例輸出：

5 8
8 6

解題方法：

以範例第1筆輸入 A = 5.699141892149156，B = 76，其對應的浮點數2進位轉10進位表示法為

0.111111111_{2} \times 2^{111111}=(2^{-1}+2^{-2}+..+2^{-8-1})\times2^{2^{6}-1}

　　將10進位等式轉成通式，另外命名m與e。

(2^{-1}+2^{-2}+..+2^{-M-1})\times2^{2^{E}-1}=(1-2^{-M-1})\times2^{2^{E}-1}=m\times2^{e}

與輸入格式兩邊取log10

\log_{10}(A\times10^{B}) = \log_{10}(m\times2^{e}) \\
\log_{10}(A) + B = \log_{10}(m) + e\times\log_{10}(2)

由於0 < A < 10，使

\log_{10}(A) < 1

，則

B=\lfloor \log_{10}(m) + e\times\log_{10}(2)\rfloor \\
A=10^{\log_{10}(m) + e\times\log_{10}(2)-B}

知道A與B的規則，且M與E的範圍不大，可以將所有M與E產生的A與B做建表。

難度為Medium

LeetCode 73 Set Matrix Zeroes

Posted on 2019-10-01 Edited on 2023-11-19 In C# , LeetCode , Level-Medium , 程式解題 , ProgramSolving

LeetCode 73 Set Matrix Zeroes 解題報告

平台：

LeetCode

題號：

73 - Set Matrix Zeroes

題目連結：

https://leetcode.com/problems/set-matrix-zeroes/

題目說明：

輸入一個m * n大小陣列matrix，若陣列中有元素為0的值，則該列與該行的元素都設為0。

解題方法：

非優化空間的方法：

建立2個HashMap，掃描matrix並分別記錄有0的Row索引與Col索引，再用這2個HashMap檢查目前[row, col]位置是否要設為0。需要O(m + n)的空間、O(mn)的時間。

優化空間的方法：

優化空間有一些步驟，如下：

  1. 尋找第0列是否有0的值，若有則標記hasCol = true
  2. 尋找第0行是否有0的值，若有則標記hasRow = true
  3. 從中間尋找有0的值，若為0則將該位置[row,col]的[row,0]與[0, col] 設為0
  4. 從中間尋找邊界為0的值，若[row, 0]或[0, col]為0則將該位置[row,col] 設為0
  5. 若hasCol = true, 將[0, 0 to n - 1]設為0
  6. 若hasRow = true, 將[0 to m - 1, 0]設為0

　　其解法在於先標記上方與左方有0元素的條件，中間與上/左方元素換成0後，最後在處理原本上/左方有0的狀況。只需要O(1)的空間、O(mn)的時間。

平台：

題號：

295 - Find Median from Data Stream

題目連結：

https://leetcode.com/problems/find-median-from-data-stream/

題目說明：

會有連續性的整數輸入，實作addNum加入整數存起來與findMedian求目前存起來整數的中位數。如果有偶數個整數，則回傳2個中間值的平均值。

解題方法：

使用2個Heap，分別是Max-Heap與Min-Heap。2個Heap的大小維持整個輸入整數數量的一半，Max-Heap的最大值為目前一半數量的最大值、Min-Heap的最小值為目前一半數量的最小值。取Median時，如果2個Heap大小一樣，取2邊Top的平均值；否則取數量較多的Top。

新增數字的時間複雜度為O(logN)、取中位數的時間複雜度為O(1)
