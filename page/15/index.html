UVa 11809 Floating-Point Numbers 解題報告

平台：

UVa Online Judge

題號：

11809 - Floating-Point Numbers

題目連結：

https://uva.onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=2909

題目說明：

題目很長，經過整理的意思是

A\times10^{B} = (2^{-1}+2^{-2}+..+2^{-M-1})\times2^{2^{E}-1}

，求M與E。輸入0e0則結束程式。

範例輸入：

5.699141892149156e76
9.205357638345294e18
0e0

範例輸出：

5 8
8 6

解題方法：

以範例第1筆輸入 A = 5.699141892149156，B = 76，其對應的浮點數2進位轉10進位表示法為

0.111111111_{2} \times 2^{111111}=(2^{-1}+2^{-2}+..+2^{-8-1})\times2^{2^{6}-1}

　　將10進位等式轉成通式，另外命名m與e。

(2^{-1}+2^{-2}+..+2^{-M-1})\times2^{2^{E}-1}=(1-2^{-M-1})\times2^{2^{E}-1}=m\times2^{e}

與輸入格式兩邊取log10

\log_{10}(A\times10^{B}) = \log_{10}(m\times2^{e}) \\
\log_{10}(A) + B = \log_{10}(m) + e\times\log_{10}(2)

由於0 < A < 10，使

\log_{10}(A) < 1

，則

B=\lfloor \log_{10}(m) + e\times\log_{10}(2)\rfloor \\
A=10^{\log_{10}(m) + e\times\log_{10}(2)-B}

知道A與B的規則，且M與E的範圍不大，可以將所有M與E產生的A與B做建表。

難度為Medium

LeetCode 73 Set Matrix Zeroes

LeetCode 73 Set Matrix Zeroes 解題報告

平台：

LeetCode

題號：

73 - Set Matrix Zeroes

題目連結：

https://leetcode.com/problems/set-matrix-zeroes/

題目說明：

輸入一個m * n大小陣列matrix，若陣列中有元素為0的值，則該列與該行的元素都設為0。

解題方法：

非優化空間的方法：

建立2個HashMap，掃描matrix並分別記錄有0的Row索引與Col索引，再用這2個HashMap檢查目前[row, col]位置是否要設為0。需要O(m + n)的空間、O(mn)的時間。

優化空間的方法：

優化空間有一些步驟，如下：

  1. 尋找第0列是否有0的值，若有則標記hasCol = true
  2. 尋找第0行是否有0的值，若有則標記hasRow = true
  3. 從中間尋找有0的值，若為0則將該位置[row,col]的[row,0]與[0, col] 設為0
  4. 從中間尋找邊界為0的值，若[row, 0]或[0, col]為0則將該位置[row,col] 設為0
  5. 若hasCol = true, 將[0, 0 to n - 1]設為0
  6. 若hasRow = true, 將[0 to m - 1, 0]設為0

　　其解法在於先標記上方與左方有0元素的條件，中間與上/左方元素換成0後，最後在處理原本上/左方有0的狀況。只需要O(1)的空間、O(mn)的時間。

LeetCode 215 Kth Largest Element in an Array 解題報告

平台：

LeetCode

題號：

215 - Kth Largest Element in an Array

題目連結：

https://leetcode.com/problems/kth-largest-element-in-an-array/

題目說明：

輸入一個整數陣列nums，求陣列中第k大的數字。

解題方法：

  1. 使用時間複雜度O(nlogn)的排序方法。
  2. 使用Heap，維護只有k個節點的Heap即可求出第k大的解，平均時間複雜度為O(nlogk)。
  3. 使用Quick Sort的原理，找第k大的數字，本篇採用使方案。依Quick Sort原理，會將陣列做部分排序，這裡從大排到小。隨機選一個點為pivot，從左邊與從右邊找出比pivot還小 / 還大 互相交換，會分成2個區塊，再判斷k位置繼續往左邊或右邊尋找。時間複雜度為O(N)。以下用圖例解釋每個步驟。

　　假設有1個輸入陣列nums為[4, 1 , 6 , 5 , 9 , 2]，left與right為最左/最右的起始位置，分別是0與5，而設置i 與 j 也分別為left 與 right。pivot的選取採用nums[left]，也就是4，而K為第2大的數字，轉成索引後變成k = 2 - 1 = 1。如圖1.

圖1

　　從左邊開始，找nums[i] <= pivot的位置；從右邊開始，找nums[j] >= pivot的位置，i與j分別停在0 與 4 的位置，並且交換，再往下個位置，如圖2.

圖2

再次找nums[i] <= pivot的位置與nums[j] >= pivot的位置，i 與 j停在 1 與 3，並且交換，再往下個位置，如圖3.

圖3

再次找nums[i] <= pivot的位置與nums[j] >= pivot的位置，i 與 j停在 3 與 2，i與j的位置已超出邊界，而k <= j ，代表k在左邊的區塊，所以將進行第2輪的排序，如圖4.

圖4

再次找nums[i] <= pivot的位置與nums[j] >= pivot的位置，i 與 j都停在1的位置，並且交換，並進到下個位置，如圖5.

此時j已經超出left，所以k不會在左邊的區塊；而k >= i，代表k將在i 與right之間，進行第3輪的排序，如圖6.

圖6

題目連結：

題目說明：

輸入只有 ( ) + - 空格 與 非負整數的數學運算式，求這運算式的結果。

解題方法：

  1. 完整的方法，需要使用Stack將中序(Infix)運算式轉成後序 (Postfix)，再解出運算結果。
  2. 網路看到有一種更快的解法，因為只有+ 跟 -，沒有其他優先序的符號，所以遇到 + 跟 - 都直接加起來，如果是有圓括 ( )的，將圓括裡面的加完再補到前面。
  3. 採用網路上另一種解法，用狀態機(Finite State Machine)，分析每一步驟要做的邏輯，如圖1. 共3種狀態，分別為Start、Number與Operatior。一開始在Start的狀態，讀取第1個字元後，將知道要往哪個狀態。Number狀態比較簡單，會一直讀數字，直到有非數字要往Operator狀態。完整的數字都存進Stack，如果前面有 + 或 - 的運算子，從Stack取出運算，再Push運算結果。Operator狀態需要分 ( ) + - 與 數字，遇到數字或( 回到Number狀態；遇到 ) 直接做+ -運算；若是+ 或 -，下一筆若是數字的話可以做+ - 運算 。詳細的方法可以參考程式碼，個人比較喜歡這種用狀態機的分析，比較清楚每一步驟的可能性。

UVa 202 Repeating Decimals

Posted on 2019-09-13 Edited on 2023-11-19 In C# , ACM , Level-Easy , 程式解題 , ProgramSolving , UVa

UVa 202 Repeating Decimals 解題報告

平台：

UVa Online Judge

題號：

202 - Repeating Decimals

題目連結：

https://uva.onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=138

題目說明：

給1個除數與被除數，計算除法後的結果。結果以小數點表示，但如果有循環小數時，則用圓括號( )顯示循環的小數點；若沒有循環小數，則用(0)表示。且輸出小數循環的長度。
　　以題目的說明，

  • 1 / 6 = 0.166666… = 0.1(6) , 循環長度為1
  • 5 / 7 = 0.714285714285…. = 0.(714285) , 循環長度為6
  • 1 / 250 = 0.004 = 0.004(0), 由於沒有循環小數點，循環長度定義為1

如果有循環小數的長度超過50個小數點，則顯示前50小數點，並用…結尾

範例輸入：

76 25
5 43
1 397

範例輸出：

76/25 = 3.04(0)
1 = number of digits in repeating cycle

5/43 = 0.(116279069767441860465)
21 = number of digits in repeating cycle

1/397 = 0.(00251889168765743073047858942065491183879093198992…)
99 = number of digits in repeating cycle

解題方法：

要計算是否有重複循環的小數點，用一個位置陣列紀錄每一次餘數的出現位置，當有餘數已被紀錄時，代表這個小數點是循環的。
　　以 5 / 7 為例，

  • 5 / 7 => 5 記錄在第1個位置 => 需進位成 50 / 7 => 商數是7，餘數為1
  • 1 / 7 => 1 記錄在第2個位置 => 需進位成 10 / 7 => 商數是1，餘數為3
  • 3 / 7 => 3 記錄在第3個位置 => 需進位成 30 / 7 => 商數是4，餘數為2
  • 2 / 7 => 2 記錄在第4個位置 => 需進位成 20 / 7 => 商數是2，餘數為6
  • 6 / 7 => 6 記錄在第5個位置 => 需進位成 60 / 7 => 商數是8，餘數為4
  • 4 / 7 => 4 記錄在第6個位置 => 需進位成 40 / 7 => 商數是5，餘數為5
  • 5 / 7 => 5 已經在第1個位置，所以找到循環的小數，長度為6

這題的輸出格式沒有寫得很清楚，有些注意的事項

  1. 如果是能整除的結果，比如 2 / 1 ，則要輸出為 2.(0)
  2. 第2行的number of digits in repeating cycle，最前面是空3格空白
  3. 每1筆測資要多1個跳行

難度為Easy
