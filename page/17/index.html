86 - Partition List

題目連結：

https://leetcode.com/problems/partition-list/

題目說明：

給1串Linked List與一個數字X，將此Linked List的節點做重新串接，小於X的節點在前面、大於或等於X的節點在後面，而節點的前後順序要與原始的一樣。以範例輸入來看，小於X = 3的有1、2、2，大於或等於X = 3的有4、3、5，所以重新串接後的Linked List = 1 -> 2 -> 2 -> 4 -> 3 -> 5 。

範例輸入：

head = 1->4->3->2->5->2, x = 3

範例輸出：

1->2->2->4->3->5

解題方法：

  1. 起點設2個空節點，分別為lessNode與greaterNode。這2個節點分別串接小於X 和 大於/等於X的節點。
  2. 再設2個ListNode指標，分別為lessPtr與greaterPtr，一開始分別指向lessNode與greaterNode。
  3. 以Head歷遍各節點，比對該節點的值與X，用lessPtr與greaterPtr連結這些節點。
  4. 最後將lessNode與greaterNode串接，完成新的LinkedList。

LeetCode 142 Linked List Cycle II

LeetCode 142 Linked List Cycle II 解題報告

平台：

LeetCode

題號：

142 - Linked List Cycle II

題目連結：

https://leetcode.com/problems/linked-list-cycle-ii/

題目說明：

給1串Linked List，檢查此Linked List是否有環 (Cycle)，若有環，回傳進入環的起點，否則回傳null。

範例輸入：

範例輸出：

tail connects to node index 1

no cycle

解題方法：

解法延續[解題] LeetCode - 141 Linked List Cycle，有兩種解法。

  • 使用Set：歷遍 Linked List ，將每個Node的記憶體位置存到Set，當有Node出現在Set，代表該那個Node是進入環的起點。否則執行到結尾點null，代表沒有環。
  • 使用Floyd’s Algorithm：
  1. 起點設2個指標，分別為Slow與Fast。
  2. 每次的循環，Slow往前1步、Fast往前2步
  3. 當Fast走到結尾點null，代表此Linked List無環；否則當Slow和Fast的位置相同時，代表此Linked List有環。
  4. 把相同位置的Node用另一個ListNode變數meetNode記住，使Slow回到起點(Head)
  5. Slow和meetNode以同樣速度往前移動，當他們走到相同位置時，則那位置是環的進入點。
  6. 有一篇文章證明這方法：Floyd’s Cycle detection algorithm Determining the starting point of cycle

難度為Medium

題號：

141 - Linked List Cycle

題目連結：

https://leetcode.com/problems/linked-list-cycle/

題目說明：

給1串Linked List，檢查此Linked List是否有環 (Cycle)。

範例輸入：

範例輸出：

true
false

解題方法：

解題有兩種解法。

  • 使用Set：歷遍 Linked List ，將每個Node的記憶體位置存到Set，當有Node出現在Set，代表該Linked List有環。否則執行到結尾點null，代表沒有環。

  3. 當Fast走到結尾點null，代表此Linked List無環；否則當Slow和Fast的位置相同時，代表此Linked List有環。

LeetCode 160 Intersection of Two Linked Lists

Posted on 2019-08-27 Edited on 2023-11-19 In C# , LeetCode , Level-Easy , 程式解題 , ProgramSolving

LeetCode 160 Intersection of Two Linked Lists 解題報告

平台：

LeetCode

題號：

160 - Intersection of Two Linked Lists

題目連結：

https://leetcode.com/problems/intersection-of-two-linked-lists/

題目說明：

給2串Linked List，求出他們的交會點。若有交會點則回傳該交會的ListNode，否則回傳null。

範例輸入：

範例輸出：

Reference of the node with value = 8

解題方法：

解題有兩種解法。

  • 使用Set：先歷遍 Linked List A，將每個Node的記憶體位置存到Set。換歷遍Linked List B，並檢查B的Node的記憶體位置有無在Set，若有在Set，代表這Node為交會點；否則沒有交會點。此方法需要額外Set的記憶體，但執行較快。
  • 使用起點對齊的方式：
  1. 由於2個Linked List的長度可能不同，事先可以計算長度。假設Linked List A與Linked List B的長度分別為lenA與lenB。
  2. lenA > lenB時，A先往前lenA - lenB步；否則B先往前lenB - lenA步
  3. 對齊後，代表這2個Linked List以同樣速度往前移動，代表能一起同時到結尾。每次移動檢查記憶體位置是否相等，若有相等，則那位置是交會點。否則一起走到結尾null，代表沒交會點。

難度為Easy

455 - Periodic Strings

題目連結：

https://uva.onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=24&page=show_problem&problem=396

題目說明：

給一個字串，求週期重複子字串的最小長度。以範例測資HoHoHo，它的Ho子字串可週期重複在第1、第3、第5的位置，而這子字串長度為2。最長的子字串是HoHoHo本身，長度為6，所以輸出答案為2。

1

HoHoHo

2

解題方法：

  1. 題目最長的字串為80，可以暴力從最小子字串求解。
  2. 符合週期重複子字串的長度一定是因數，所以可以省略非因數的解，當檢查到第Len/2的長度仍沒解時，則最小長度只會是題目本身。

5 OOXXOXXOOO OOXXOOXXOO OXOXOXOXOXOXOX OOOOOOOOOO OOOOXOOOOXOOOOX

10
9
7
55
30

解題方法：
