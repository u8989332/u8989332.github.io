UVa 1594 Ducci Sequence

Posted on 2019-12-22 Edited on 2023-11-19 In C# , ACM , Level-Easy , 程式解題 , ProgramSolving , UVa

UVa 1594 Ducci Sequence 解題報告

平台：

UVa Online Judge

題號：

1594 - Ducci Sequence

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=4469

題目說明：

給1組數字序列，包含N個數字，這1組序列不斷計算成新的序列，求這序列是否會重複出現或者全都是0。若有重複出現輸出LOOP、全都是0輸出ZERO。
計算新序列的規則：
假設原本序列為

[a1,a1, …., an] ，

新的序列為

 [a1-an, a1-a2, …., an-1-an] 題目保證重複出現或全都是0的計算步驟不超過1000次。

範例輸入：

4
4
8 11 2 7
5
4 2 0 2 0
7
0 0 0 0 0 0 0
6
1 2 3 1 2 3

範例輸出：

ZERO
LOOP
ZERO
LOOP

解題方法：

這題有些解法：

  1. 暴力法：由於題目保證每個序列計算步驟不超過1000次，當迴圈超過1000次仍沒有全是0的狀況，則輸出LOOP，否則會是ZERO。
  2. Hash：紀錄每次計算出的新序列，是否出現在Hash，有的話則是LOOP。
  3. Floyd’s 檢測Cycle：可以把重複出現的檢測方法轉換成Floyd’s的檢測Cycle演算法，速度可以比前兩者還要快，可以參考Linked List版的設計　[解題] LeetCode - 141 Linked List Cycle

　　
難度為Easy

1593 - Alignment of Code

題目說明：

給N列的程式碼字串，每列程式碼包含空格和其他符號組合的單字，求重新排列的程式碼字串。排列的規則為，取第K行最長單字的長度Lk，同K行的單字都要靠左並補上空白，使第K行的長度都為Lk。每行單字中間用1個空格分隔，每列的最後一個單字不必補空白，直接跳行。
　　以範例輸入為例，第1行的單字包含start:、stop:、s:與c:，最長單字start:的長度為6，排列後的結果如範例輸出。

start: integer; // begins here
stop: integer; // ends here
 s: string;
c: char; // temp

範例輸出：

start: integer; // begins here
stop: integer; // ends here
s: string;
c: char; // temp

解題方法：

使用C++的stringstream做每列單字的切割，計算每一行的最長單字的長度，最後用iomanip函式庫的left與setw靠左輸出。

題目說明：

1個地圖上有N個矩形建築物，每個建築物有X、Y、寬度、深度與高度，X是朝向南面的左下角座標、Ｙ是朝向西面的左下角座標。

　　如題目的示意圖圖1，左半邊是俯視圖，右半邊是從南面地上看建築物的樣貌，從最西方開始數，建築物順序分別是5 9 4 3 10 2 1 14。

解題方法：

原先想採用暴力的方法，窮舉所有X的範圍 0 至 最大X，但是X有可能很大，這樣會沒效率。看了劉汝佳大師的解法，基於離散化將X的範圍縮小，縮小的方式如下述圖解。

　　假設建築物的排列如圖2，有3棟，從南往北分別是1 、2、3。一開始做排序，先從X最小的排，如果X有相等再從Y最小的排，所以排序後結果是1、3、2。

圖2

接著先列出所有X的組合，包含建築物最西邊與最東邊的X座標，初始順序是0、60、20、60、0、70，把它們排序且除去重複後，剩下0、20、60、70。
　　X相鄰區間的點，要麼會被看見、要麼不會被看見，所以只需要取區間內任一點作為檢查該建築物的可見性。
　　取X兩兩相鄰元素平均值，作為判斷此建築物是否能被看見的標準，如圖3，平均值MX分別是10、40、65。

圖3

由於有對建築物做排序，第1個會被看見的一定是第1個元素，即為ID = 1的建築物。第2個元素是ID = 3，需要藉由MX做判斷。
　　MX = 10時，由於ID = 1的高度擋住了ID = 3的高度，所以此區間看不見，如圖4

圖4

MX = 40時，由於ID = 1與ID = 2的高度擋住了ID = 3的高度，所以此區間看不見，如圖5

圖5

MX = 65時，無建築物擋住了ID = 3，所以此區間可看見，如圖6

圖6

因此第2個可看見的建築物為ID = 3。最後ID = 2的建築物從MX的判斷無法被看見，所以不包含它。

題目說明：

這題細節超多，完整題目還是請看原文，這邊做些整理。

計算高爾夫球的獎金計算，總獎金會有1筆，並有70種分配的比例。有n位選手，每位選手有獨自的名字和4種回合的分數。如果是業餘選手，名字最後字元是米字號 * 。4種回合的分數，如果中間第N回合失去資格，值以DQ呈現。

如果有多個同分的選手，則獎金比例採用平均，比如第2和第3位選手同分，則這2位選手都得到 (Percent-2nd + Percent-3rd) * 總獎金 / 2。
　　職業選手和業餘選手都要顯示排名，如果職業選手有同分的狀況，排名後面要加個T字。
業餘選手不能得到獎金，只顯示排名。
　　如果有第3或第4回合DQ的選手，則要輸出在最後面且不會有獎金。

UVa 1592 Database

Posted on 2019-11-21 Edited on 2023-11-19 In C# , ACM , Level-Medium , 程式解題 , ProgramSolving , UVa

UVa 1592 Database 解題報告

平台：

UVa Online Judge

題號：

1592 - Database

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=4467

題目說明：

有n列資料，每列資料有m個欄位的值。m個欄位值用逗點 , 分隔組成1列資料。求這n列資料是否為PNF，PNF的定義沒有出現2個不同列r1 與 r2在2個不同欄位c1與c2，同欄位有相同的值，也就是table[r1][c1] = table[r2][c1] 且 table[r1][c2] = table[r2][c2]。
　　如果是PNF，輸出YES；如果非PNF，則輸出NO，並輸出r1、r2、c1、c2。

範例輸入：

3 3
How to compete in ACM ICPC,Peter,peter@neerc.ifmo.ru
How to win ACM ICPC,Michael,michael@neerc.ifmo.ru
Notes from ACM ICPC champion,Michael,michael@neerc.ifmo.ru
2 3
1,Peter,peter@neerc.ifmo.ru
2,Michael,michael@neerc.ifmo.ru

範例輸出：

NO
2 3
2 3
YES

解題方法：

每1列每1欄位的值可存在2維的Table，並用Map紀錄第j欄的字串出現在哪幾列。當第i筆的字串有出現在Map時，需要檢查同列且其他欄位的字串，是否他們的Map對應的列數有交集，交集可以用STL的set_intersection。
　　這題有個速度優化技巧，比如m如果只有1，一定是PNF，因為它定義c1 != c2。

POJ 3617 Best Cow Line

Posted on 2019-11-19 Edited on 2023-11-19 In C# , ACM , Level-Easy , 程式解題 , ProgramSolving , POJ

POJ 3617 Best Cow Line 解題報告

平台：

PKU Online Judge

題號：

3617 - Best Cow Line

題目連結：

http://poj.org/problem?id=3617

題目說明：

給予N個字元，每個字元由A…Z組成。每1次取這N個字元的最前面或最後面的1個字元，來形成新的字串，經過N次的取字元後，求形成最小字典順序的字串。
　輸出字串時每1列最多只能印80字元，多的繼續往下1列印。

範例輸入：

6
A
C
D
B
C
B

範例輸出：

ABCBCD

解題方法：

此題為貪婪法，貪婪的策略不可以只取當前最小的前面或後面，而是由前面數來的字串與後面數來的字串做字典大小比對，取字典最小的端點字元。
比如測資ACDBCB 與 反向 BCBDCA，此時取A
接著CDBCB與BCBDC，此時取B
接著CDBC與CBDC，此時取CBDC的C
接著CDB與BDC，此時取B
接著CD與DC，此時取C
最後是D，新形成的字串為ABCBCD

UVa 136 Ugly Numbers 解題報告

平台：

UVa Online Judge

題號：

136 - Ugly Numbers

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=72

題目說明：

若1個數字，它只能被2、3、5整除的話，則它是Ugly Number，求第1500個Ugly Number

範例輸入：

沒有輸入

範例輸出：

The 1500’th ugly number is 答案.

解題方法：

需使用Priority Queue和Set ：

LL = long long;

  • priority_queue<LL, vector, greater > pq：用來存目前所有的Ugly Number，Top代表當前最小Ugly Number
  • set s：紀錄已經被計算過的Ugly Number，若已被算過，則不加入pq。

　 一開始在pq、s先填入1值，接著從pq不斷取top、pop算到1500次的top為題目的解。

  1. 第1次pq的top且pop的是1，1 * 2 = 2、1 * 3 = 3、 1 * 5 = 5 , 2 、3 、5都不在s裡，所以都push到pq。
  2. 第2次pq的top且pop的是2，2 * 2 = 4、2 * 3 = 6、 2 * 5 = 10，4、6、10都不在s裡，所以都push到pq。
  3. 第3次pq的top且pop的是3，3 * 2 = 6、3 * 3 = 9、 3 * 5 = 15，6已在s裡面，而9、15都不在s裡，所以只push 9、15到pq。
  4. 第4次pq的top且pop的是4，4 * 2 = 8、4 * 3 = 12、 4 * 5 = 20，8、12、20都不在s裡，所以都push到pq。
  5. 第5次pq的top且pop的是5，5 * 2 = 10、5 * 3 = 15、 5 * 5 = 25，10、15已在s裡面，而25不在s裡，所以只push 25到pq。
  6. 經過前述的邏輯，到第1500次pq的top且pop的是859963392

Priority Queue的Push與Pop都是O(logN)的時間複雜度，所以整體需O(NlogN)。

題目說明：

有t個隊伍，每個隊伍有各自成員，成員都有獨立的編號。他們會各自去排一個大隊伍，若大隊伍裡面沒有同隊伍的人，則該員排在該大隊伍的最後面；否則可以直接插隊，排到大隊伍裡面同隊成員的最後面。
