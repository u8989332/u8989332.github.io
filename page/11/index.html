UVa 1596 Bug Hunt 解題報告

平台：

UVa Online Judge

題號：

1596 - Bug Hunt

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=24&page=show_problem&problem=4471

題目說明：

檢查一段程式碼是否有BUG，程式碼只包含陣列的賦值和宣告。陣列宣告是該變數名稱與它的size，且size只會是純數字；但賦值的狀況不同，等於的右方會以純數字或者一個（或多個）陣列變數，等於的左方會以一個（或多個）陣列變數。
　　求程式碼執行過程第一次會出錯的行數，比如超出索引值、變數尚未被賦值的那一行；假如都沒問題，則輸出0。 　　

範例輸入：

a[3]
a[0]=a[1]
.
x[1]
x[0]=x[0]
.
a[0]
a[0]=1
.
b[2]
b[0]=2
b[1]=b[b[0]]
b[0]=b[1]
.
g[2]
G[10]
g[0]=0
g[1]=G[0]
.
a[2147483647]
a[0]=1
B[2]
B[a[0]]=2
a[B[a[0]]]=3
a[2147483646]=a[2]
.
.

範例輸出：

2
2
2
3
4
0

解題方法：

　　模擬題，主要由2個資料結構：

  1. map<string, int> variable_def : 陣列變數名稱與它的size
  2. map<MyVar, int> variable_val : 陣列變數與索引對應的值, 而他的型別是struct

struct MyVar {
string name;
int index;

bool operator< (const MyVar &x) const {
return name < x.name name == x.name && index < x.index;
}
};

使用operator< 作為map的key compare

  • 當沒有 = 符號，代表是宣告，將[]的數值與前面名字存到variable_def。
  • 當有=符號，代表是賦值，分左方與右方的檢查：左方只需判斷變數是否有被定義和超出索引，而右方需判斷是否為純數字或變數。
  • 變數用遞迴往[]的裡面找，直到最後無[]，並回傳內部的變數與索引值是否正確。
  • 最後左方與右方都正確，再存進variable_val裡。

難度為Medium

UVa 230 Borrowers

UVa 230 Borrowers 解題報告

平台：

UVa Online Judge

題號：

230 - Borrowers

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=24&page=show_problem&problem=166

題目說明：

一開始圖書館架上會有好幾本書，書先以作者名做字典排序，如果有相同名字，再以書名做字典排序。書名保證不重複。
　　接著有三種指令，包含

  • BORROW 書名 : 從書架上取下該本書
  • RETURN 書名 : 將被借的書歸還(尚未放在書架)
  • SHELVE : 將目前被歸還的書籍，放回到書架上。被放回書架上要依照前者的排序顯示放回去的狀況。如果是最前面的書，則輸出

Put 書名 first
　　 否則輸出該本書的前一本 
Put 當前的書名 after 前一本書名

　　 假如SHELVE時沒任何書被歸還，則不用輸出。最後遇到END結束該題。

範例輸入：

“The Canterbury Tales” by Chaucer, G.
“Algorithms” by Sedgewick, R.
“The C Programming Language” by Kernighan, B. and Ritchie, D.
END
BORROW “Algorithms”
BORROW “The C Programming Language”
RETURN “Algorithms”
RETURN “The C Programming Language”
SHELVE
END

範例輸出：

Put “The C Programming Language” after “The Canterbury Tales”
Put “Algorithms” after “The C Programming Language”
END

解題方法：

　　模擬題，資料結構分別使用

  1. map<string, string> titleToAuthor : 書名對應作者名
  2. vector shelve : 書架上的書
  3. vector returnBooks : 被歸還的書, 而他的型別是struct

struct ReturnBook{ string title; string author; };

  1. 一開始將shelve做排序，當有書被借走時，可用lower_bound 二分搜尋去找該書並把它移除。
  2. 當有書歸還時，都先Push至returnBooks。
  3. 遇到Shelve指令時，將returnBooks作排序，從第1筆開使二分搜尋找shelve對應的位置，如果是等於shelve的begin()， 代表它是最前面的書，要輸出

Put 書名 first

，否則二分搜尋找的那一本會是它前一本書，要輸出

Put 當前的書名 after 前一本書名

難度為Easy

POJ 3253 Fence Repair

Posted on 2020-02-28 Edited on 2023-11-19 In C# , ACM , Level-Easy , 程式解題 , ProgramSolving , POJ

POJ 3253 Fence Repair 解題報告

平台：

PKU Online Judge

題號：

3253 - Fence Repair

題目連結：

http://poj.org/problem?id=3253

題目說明：

給予一大塊木板，要切成N小塊的子木板，每個子木板有各自的Li (i = 1 … N)長度，每次切一刀成兩半時，需花費的成本是被切之前的長度。求切割最小的花費成本。
比如範例輸入N = 3，要的子木板長度各為8、5、8，而切第一刀之前(分成13 + 8)的長度是21，切第二刀之前的長度是13 (分成8 + 5)，所以最小的成本是21 + 13 = 34。

範例輸入：

3

8

5

8

範例輸出：

34

解題方法：

　　此題為貪婪法，且是經典的霍夫曼編碼應用。霍夫曼編碼可以參考演算法筆記的講解，其方法是依據要壓縮對象的字元出現頻率，將它們編碼後，使對象壓縮後的長度是最小的。
　　原本霍夫曼編碼是以出現頻率最小開始編碼，換成這題是從長度最小的木板。實作方式可用Priority Queue，每次取最小長度的2個木板，合成後的長度做累加，再放回Priority Queue，直到所有木板都合成完，則最後累加的長度為答案。

難度為Easy

POJ 3069 Saruman's Army

POJ 3069 Saruman’s Army 解題報告

平台：

PKU Online Judge

題號：

3069 - Saruman’s Army

題目連結：

http://poj.org/problem?id=3069

題目說明：

給予N個石頭，每個石頭有各自的X座標位置(0 到 1000)，再給予一個長度R，每個石頭能向它位置的左方/右方延伸R單位，涵蓋其他的石頭，這包覆稱為palantir。求palantir最少的使用量。
　　以範例第2筆測資，4個palantir最少的位置如圖1.

圖1

範例輸入：

0 3
10 20 20
10 7
70 30 1 7 15 20 50
-1 -1

範例輸出：

2

4

解題方法：

　　此題為貪婪法，貪婪的策略為先從最左方的石頭A開始，往右方尋找R單位內最遠的石頭B，找到該石頭B後，再往右方尋找R單位內最遠的石頭C，而B就會是一個palantir。下一次的起點是離C最近的右方石頭開始前述的判斷。
　　以範例測資第2筆，R = 10，首先從x = 1石頭，離它x右邊 1 + 10 = 11以內的最右邊石頭為x = 7，接著再從 7 + 10 = 17 以內最右邊的石頭為x = 15，此時x = 7為第1個palantir。
　　下一個回合的石頭是x = 20，離它x右邊 20 + 10 = 30以內的最右邊石頭為x = 30，接著再從 30 + 10 = 40以內最右邊的石頭找不到，此時x = 30為第2個palantir。
下一個回合的石頭是x = 50，離它x右邊 50 + 10 = 60以內的最右邊石頭找不到，此時x = 50為第3個palantir。
下一個回合的石頭是x = 70，離它x右邊 70 + 10 = 80以內的最右邊石頭找不到，此時x = 70為第4個palantir。
　　所以第2個測資的答案為4。

難度為Easy

UVa 12100 Printer Queue 解題報告

平台：

UVa Online Judge

題號：

12100 - Printer Queue

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=3252

題目說明：

會有N個列印工作，每個工作會有優先序(Priority)，1是最小、9是最大。如果在列隊前方的工作的優先序，比其他工作優先序還小，則前方的工作要排到後方。如果列隊只剩它優先序最大，則輪到它列印。
　　題目有另一個輸入m，代表原始列隊裡的工作位置(0 到 N - 1)，求何時輪到第m個工作被列印。
　　以範例輸入第2筆測資，輸入4個列印工作、指定m = 2(工作優先序 = 3的那一筆)，一開始工作優先序1發現列隊有比它大的工作，則排到後方，變成2 3 4 1。接著優先序2發現有比它大的工作，則排到後方，變成3 4 1 2。接著優先序3發現有比它大的工作，則排到後方，變成4 1 2 3。接著優先序4是列隊最大的優先序，則它可列印，剩下1 2 3。重複前面的規則，優先序3的工作是第2個列印，所以輸出答案2。

範例輸入：

3
1 0
5
4 2
1 2 3 4
6 0
1 1 9 1 1 1

範例輸出：

1
2
5

解題方法：

　　單純的模擬題，建1個Queue，這N個工作依序放在Queue並依前述規則輸出。另外需建1個優先序陣列，紀錄優先序1~9的數量剩幾個，才能知道目前要列印的工作是不是它。
難度為Easy

UVa 10391 Compound Words 解題報告

平台：

UVa Online Judge

題號：

10391 - Compound Words

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=1332

題目說明：

會有N個字串，求這N個字串任選2個並連接成的新字串，是否出現在N個字串裡。

範例輸入：

a
alien
born
less
lien
never
nevertheless
new
newborn
the
zebra

範例輸出：

alien
newborn

解題方法：

  • 建1個Set，這N個字串都放在Set裡，然後歷遍Set每個字串，依據該字串的長度Len切成Len - 1種組合的2個子字串，看這些子字串是否都存在Set，如果都存在則它就是一個解。
  • 比如alien，可以切成{a, lien}, {al, ien}, {ali, en}, {alie, n} 4種組合，第1個組合有符合即輸出。

難度為Easy

UVa 1595 Symmetry 解題報告

題號：

1595 - Symmetry

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=4470

題目說明：

給N個2維的座標，座標包含X與Y值，求是否能找到一條垂直線，它能劃分左半邊與右半邊，使左半邊的座標對稱於右半邊的座標。有這一條垂直線輸出YES，否則NO。

範例輸入：

3
5
-2 5
0 0
6 5
4 0
2 3
4
2 3
0 4
4 0
0 0
4
5 14
6 10
5 10
6 14

範例輸出：

YES
NO
YES

解題方法：

  • 一開始的解法是對座標做2種排序，分別從X從小到大與X從大到小，再依X最右邊與最左邊的2個座標取X的平均值，一個一個比對。但是執行速度不快，需0.02秒。
  • 參考高手暴力法．．．https://blog.csdn.net/acvay/article/details/43015507，只要一個一個比對就好，連排序都不用，速度是0.00秒….
  • 由於要取X的平均值，要先將X都乘以2，平均才不會有奇數的問題。

　　輸入有一種測資是

1
3

這代表會有多個點是同1個座標，所以要此測資為YES。

1593 - Alignment of Code

題目連結：

https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=4468

題目說明：

給N列的程式碼字串，每列程式碼包含空格和其他符號組合的單字，求重新排列的程式碼字串。排列的規則為，取第K行最長單字的長度Lk，同K行的單字都要靠左並補上空白，使第K行的長度都為Lk。每行單字中間用1個空格分隔，每列的最後一個單字不必補空白，直接跳行。

stop: integer; // ends here
 s: string;
c: char; // temp

範例輸出：

start: integer; // begins here
stop: integer; // ends here
s: string;
c: char; // temp

解題方法：

使用C++的stringstream做每列單字的切割，計算每一行的最長單字的長度，最後用iomanip函式庫的left與setw靠左輸出。
