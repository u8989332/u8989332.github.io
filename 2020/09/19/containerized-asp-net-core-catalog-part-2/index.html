1. 新增繼承DbContext的類別

　在CatalogApi專案新增資料夾Data，新增CatalogContext類別，並繼承DbContext，要與DB存取資料都是以此物件。

　　前述的CatalogType和CatalogItem兩個Entity加入至CatalogContext的DbSet，才能對這兩個做CRUD的功能。 　　在這使用Fluent API，可以宣告資料庫的表格規範，包含欄位的限制、Relation的設計等。

2. 新建Docker Compose文件，並啟用Sql Server

　　在方案的根目錄，建立docker-compose.yml，並加入至方案。yml的內容如下，其中有幾個參數：

  • networks：將資料庫規範在backend的網路區域，後續其他系統會歸類在其他區域
  • image：從Docker Hub要安裝的SQL Server Image，預設用目前最新的
  • ports：本機端的Port 1445映射Docker container的1433 port，由於本機已經裝過SQL Server，所以要用另外的Port 1445
  • container_name：建立本專案SQL Server的container名稱
  • environment：啟用SQL Server必須給定的參數有ACCEPT_EULA和SA_PASSWORD，前者參數必須指定Y，否則不給用；後者是SQL Server預設管理者SA的密碼

3. 設定appsettings.json的連線字串

　　在appsettings.json增加4個屬性

“DatabaseServer”: “localhost,1445”,

“DatabaseName”: “CatalogDb”,
“DatabaseUser”: “sa”,

“DatabasePassword”: “JustTest!”

將連線字串的主要參數拆解，可以區分用VS和Dokcer的測試參數。
